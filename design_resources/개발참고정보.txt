-------------------------------------------------------
## Agent의 파일 변경 감지 방식
-------------------------------------------------------

1. 자동 감지/체크는 없습니다
수동 변경을 자동으로 감지하거나 추적하지 않습니다.
변경사항을 자동으로 되돌리거나 덮어쓰지 않습니다.

2. 파일을 읽을 때만 확인
작업 중 파일을 읽으면 그 시점의 저장된 상태를 봅니다.
예: read_file로 읽으면 현재 저장된 내용이 표시됩니다.

3. 컨텍스트 유지
대화 중에 읽은 파일 내용은 기억하지만, 파일이 외부에서 변경되면 자동으로 갱신되지 않습니다.
새 작업을 시작하거나 파일을 다시 읽으면 최신 상태를 확인합니다.

* 권장 사항

수동 수정 후 작업을 이어갈 때

변경 사항을 요청에 명시
예: "폰트를 Noto Sans로 변경했고, primary 색상을 #0066CC로 바꿨어요"

관련 파일을 다시 읽도록 요청
예: "globals.css를 확인하고 작업해주세요"

변경 사항을 문서화
예: "tailwind.config.ts의 색상 변수를 수정했어요"

* 예시 시나리오
사용자: "globals.css에서 폰트를 수정했어요. 이제 버튼 스타일을 업데이트해주세요"
→ Agent는 globals.css를 읽어서 현재 폰트 설정을 확인하고 작업합니다.사용자: "아무 말 없이 버튼 스타일만 업데이트해주세요"
→ Agent는 이전에 본 globals.css 내용을 기반으로 작업할 수 있지만,   수동으로 변경한 내용이 있다면 반영되지 않을 수 있습니다.

* 결론
- 수동 변경은 저장되면 파일을 읽을 때 확인됩니다.
- 자동으로 추적하거나 복원하지는 않습니다.
- 작업을 이어갈 때 변경 사항을 알려주거나 관련 파일을 다시 읽도록 요청하세요.
