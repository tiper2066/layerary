## 핀터레스트 스타일 UI


* CSS Columns 방식이 아니라, JavaScript를 이용한 '절대 위치(Absolute Positioning)' 방식


1. 핀터레스트가 사용하는 방식: JS 기반 절대 좌표 계산

핀터레스트처럼 수만 개의 이미지를 무한 스크롤로 보여줘야 하는 서비스는 단순히 CSS만으로 해결하기 어렵습니다.

동작 원리: 자바스크립트가 각 이미지의 높이를 계산한 뒤, top과 left 좌표를 직접 지정하여 배치합니다.

사용 이유: * 순서 유지: CSS Columns 방식은 데이터가 '위에서 아래로' 먼저 쌓이지만, 핀터레스트는 새로운 콘텐츠가 항상 '왼쪽에서 오른쪽' 순서로 빈 공간에 채워져야 합니다.

무한 스크롤: 사용자가 화면을 내릴 때마다 새로운 카드를 기존 레이아웃을 깨뜨리지 않고 가장 짧은 열(Column) 아래에 딱 붙여야 하기 때문입니다.


2. 앞서 설명한 CSS Columns 방식과의 차이점

구분,CSS Columns (추천드린 방식),Pinterest 방식 (JS 기반)
구현 난이도,매우 쉬움 (클래스 한 줄),높음 (계산 로직 필요)
데이터 순서,세로 방향 (위 → 아래),가로 방향 (좌 → 우)
성능,브라우저 자체 렌더링으로 빠름,이미지 로드 후 계산 작업 필요
추천 상황,"포트폴리오, 일반 갤러리","무한 스크롤, 방대한 양의 데이터"


3. 실무에서는 어떻게 구현하나요?

직접 모든 좌표를 계산하는 것은 매우 복잡하기 때문에, 실무에서 핀터레스트 같은 UI를 만들 때는 잘 만들어진 라이브러리를 주로 사용합니다.

React 사용 시: react-masonry-css 또는 react-plock

Vanilla JS 사용 시: Masonry.js (가장 유명함)

직접 구현하고 싶을 때: 각 열(Column)의 높이 합계를 배열로 관리하며, 새로운 카드를 추가할 때마다 **'가장 높이가 낮은 열'**에 카드를 배치하는 로직을 작성합니다.
